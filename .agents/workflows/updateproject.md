# 项目更新工作流

此工作流用于总结当前改造内容，并调用版本升级接口进行登记。

## 使用方法

```bash
/updateproject
```

## 执行步骤

### 1. 检查代码变更

首先检查当前的代码变更情况：

```bash
git status --short
```

### 2. 生成变更总结

使用 AI 分析代码变更，生成简洁的变更总结。总结应包括：
- 主要功能变更
- 新增的模块或服务
- 修复的问题
- 性能优化
- 文档更新

总结应控制在 100 字以内，突出核心改动。

### 3. 调用版本升级接口

使用生成的总结作为版本描述，调用版本升级脚本：

```bash
bun run scripts/upgrade-version.ts --description "<变更总结>"
```

### 4. 确认升级结果

检查版本升级是否成功，并显示：
- 旧版本号
- 新版本号
- 升级时间
- 版本描述

### 5. 更新文档（可选）

如果有重大功能更新，提示用户更新以下文档：
- README.md
- ARCHITECTURE.md
- CHANGELOG.md（如果存在）

## 注意事项

1. **确保 API 服务运行**：版本升级需要 API 服务在 http://localhost:3000 运行
2. **提交代码**：建议先使用 `/mygit` 提交代码，再执行此工作流
3. **版本规则**：
   - 同日升级：序号递增 (001 → 002)
   - 跨日升级：序号重置为 001
   - 最大序号：999

4. **总结质量**：
   - 简洁明了，突出核心功能
   - 使用专业术语
   - 避免冗长描述
   - 控制在 100 字以内

## 相关命令

- `/mygit` - AI 智能提交代码
- `/savesession` - 保存会话总结
- `bun run scripts/upgrade-version.ts --help` - 查看版本升级帮助

## 工作流脚本

此工作流会自动执行以下操作：

1. 检查 git 状态
2. 使用 AI 生成变更总结
3. 调用版本升级 API
4. 显示升级结果
5. 提示后续操作

所有步骤都会有清晰的日志输出，方便追踪执行过程。
