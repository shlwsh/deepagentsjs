# Git 智能合并工作流

这个工作流会自动完成以下操作:

1. 检测当前是否在 Git 仓库内，如果本地有新修改则使用 `git stash` 暂存保护
2. 借用 `git pull --rebase` 拉取线上的最新更改
3. 遇到冲突的文件，尝试交给内置大语言模型去智能阅读多方代码变动逻辑并生成合理正确的整合代码，并自动 `git add`
4. 对于因为 `rebase` 引起的部分，在解决完毕后执行 `git rebase --continue`；对 `stash pop` 则无缝恢复你的工作现场。

## 使用方法

通常在你想要获取云端最新代码更新而本地又恰好未加塞的时候调用：

```bash
bun run mymerge
```

## 工作流程

1. **检查 Git 缓存与暂存** - 若直接存在 uncommitted 变化，防丢本地记录
2. **拉取动作** - 以远端分支覆盖进行 `pull --rebase`
3. **冲突切入 (LLM)** - 对每一个报出 `UU`/双重修改等标识的文件进行 `conflict-resolver.ts` 模型唤醒
4. **输出验证** - 获取干净的代码片段直接写入项目文件
5. **继续流程** - 解决全部矛盾以后放行 Git 工具自行完成后续组装。

## 注意事项

- 因为涉及 LLM 阅读冲突的分析时间，对于包含几十个文件大冲突的网络请求也许需要一定耐心；
- 如果你遇到十分特殊或大模型能力不足导致语法缺失的文件解决结果被识别出，脚本会退出中断，你可以选择手动修复遗留冲突或直接走标准的 `git rebase --abort` 取消。
- 当执行一切完全成功以后，若无更多修改你顺手可以使用 `bun run mygit`。
